# Issue #37: スマホ操作対応

## 概要
スマートフォンのタッチ操作でゲームをプレイできるようにする。

## 目的
QRコード接続（#36）と組み合わせて、スマホからゲームに参加できるようにする。

## ゴール
- タッチ入力による移動操作
- レスポンシブなキャンバスサイズ
- モバイルフレンドリーなUI

## 変更箇所

### クライアント側
- `client/src/input.ts`: タッチイベントハンドラの追加
  - バーチャルジョイスティック（左半分: 移動）
  - touchstart/touchmove/touchend でキーマスクを生成
  - 射撃は自動（既にサーバー側で最寄り敵に自動射撃）
- `client/src/renderer.ts`: キャンバスのレスポンシブ対応
  - viewport metaタグの設定
  - デバイスサイズに応じたスケーリング
- `client/src/game.ts`: バーチャルジョイスティックUIの描画
- `client/index.html`: viewport設定、タッチ操作のスクロール無効化

### UIデザイン
- 画面左半分: バーチャルジョイスティック（半透明の円形UI）
- タッチした位置を基点に、ドラッグ方向で移動方向を決定
- デッドゾーン付きで微小な入力を無視

## テスト
- スマホ実機でタッチ操作が正しく動作することを確認
- Chrome DevToolsのモバイルエミュレーションで動作確認
- キャンバスが画面サイズに合わせてリサイズされることを確認
- 既存のキーボード操作に影響がないことを確認

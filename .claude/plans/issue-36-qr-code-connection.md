# Issue #36: QRコードで接続できるようにする

## 概要
同じネットワーク上のデバイスからQRコードを読み取ってゲームに接続できるようにする。

## 目的
LAN内でのマルチプレイ参加を簡単にする。URLを手打ちする手間を省く。

## ゴール
- サーバーのアドレスをQRコードとして表示
- QRコードをスキャンするとブラウザでゲームクライアントが開く

## API
### 新規エンドポイント
- `GET /api/connection-info` — サーバーのローカルIPとポートを返す
  - レスポンス: `{ "url": "http://192.168.x.x:3000", "ws": "ws://192.168.x.x:3000/ws" }`

### サーバー側の変更
- `server/cmd/main.go` または `server/router.go`: エンドポイント追加
- ローカルIPアドレスの自動検出（`net.InterfaceAddrs()`）

## インフラ
- クライアント側でQRコード生成ライブラリを使用（例: `qrcode.js`）
- QRコードの内容: `http://{localIP}:{port}`
- ゲーム画面のUIにQRコード表示ボタンを追加

## テスト
- QRコードが正しいURLを含んでいることを確認
- 別デバイスからQRコード経由で接続できることを確認
- 複数ネットワークインターフェースがある場合の挙動確認

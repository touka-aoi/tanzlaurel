# Issue #39: 未ログイン時の編集モードを削除する

## 概要
現在、未ログインユーザーでも記事の作成・編集が可能になっている。ログイン済みユーザーのみが編集できるように制限する。

## 目的
ブログの記事品質を保つため、認証済みユーザーのみに編集権限を付与する。

## ゴール
- 未ログインユーザーは閲覧のみ可能
- ログイン済みユーザーのみが記事の作成・編集・削除が可能
- WebSocket接続（リアルタイム編集）も認証必須にする

## API
### 変更が必要なエンドポイント
- `POST /api/entries` — 認証ミドルウェアを追加（記事作成）
- `GET /api/ws` — 既にチケット認証あり。未認証時のフォールバック接続を禁止する
- `POST /api/ws-ticket` — 変更なし（既に認証必須）

### サーバー側の変更
- `web/server/router.go`: `/api/entries` POSTに認証ミドルウェアを適用
- `web/server/handler/ws.go`: チケットなしのWebSocket接続を拒否する
- `web/server/handler/entry.go`: CreateEntry に認証チェックを追加

## テスト
- 未認証状態で `POST /api/entries` → 401を返すことを確認
- 未認証状態でWebSocket接続 → 拒否されることを確認
- 認証状態で記事作成・編集が正常に動作することを確認
- フロントエンドで未認証時にComposeBox・編集UIが非表示になることを確認
- E2Eテスト (`web/cockpit/e2e/`) の更新

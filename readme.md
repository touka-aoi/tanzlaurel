### ギモン
リクエスト／レスポンス（req/res）型の並行処理サーバーと、シングルループ構成（単一スレッドで逐次処理）のサーバーでは、ユーザー間が相互作用が多い場合どちらの方が効率的にリクエストを処理できるか？

### カセツ？
ユーザー間の相互作用が行われると、共有データへのアクセスやロック競合が発生し、スレッド間で同期コストが増加する。そのため、シングルループ構成の方が効率的なのではないか

### ケイソク
1. 効率的とは単位時間当たりに処理できたreq/res回数とする。
2. １リクエスト１ゴルーチン(スレッド)の並行処理(net/http)、シングルループ構成のサーバーを両方立て、負荷をかけることで計測する。アプリケーションロジック部分は共通。

### ギモン2
I/Oは並行にしてアプリケーションを単ループで処理したらI/Oの差だけになってその場合は、受付量が多い並行の方が強いのでは

### ギモン3
シングルループはCPUの処理能力に依存(垂直スケーリングしかできない)ので、超大人数の時はどうするんだろうか. なんか並行でうまくやる方法がありそう？
